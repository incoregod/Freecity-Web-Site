Books Instalation

ESX Framework

Creating Books:

  • Create your book items on your Database items table;
  • Add to shared.lua file in Config.Items the respective Books
  • The name in [ ] must match your items table item and image name
  • Create your book Covers and pages by designing them using for example photoshop or other software. Check the example we made
  • Put your created images inside of html/pages folder with the name of book item plus the page number Example:guide1(page1) guide2(page2)
  • You can add as many books and pages as you want
  • Last... make sure that if you're running a default framework that you edit the triggers

Capture the Flag Instalation

ESX & QB-CORE Framework

This resource uses escrow Fivem protection. All needed modifications can be made on shared-server / shared-client and shared.lua

We recomend the use of polyzones, but our script is full compatible without them

Dependencys:

  • Gamebuild >= 2372 If you want to run our pre-configured map: TunnerMeetup

Instalation:

  • Please take a look at the file shared.lua and take the following steps in order to get this script up and running
  • Add your revive triggerEvent in order to revive players. Config.ReviveEvent = 'esx_ambulancejob:revive'

    • Targeting system or Blip system, your choice

  • Decide if you want to use a targeting system or a blip system using the vars:

    Config.spawnMarker

    Config.useTargetSystem

  • Choose if you want to spawn a ped at the startLocation (if using targeting system it's mandatory)

    • Clothes, Maps & GameWeapons

  • Configure the clothes, that the players will be able to choose when in lobby
  • All Clothes must be created in shared.lua file following the examples already created there.
  • Configure the Maps and GameWeapons the HostLobby Leader will choose from
  • All Maps and GameWeapons must be created in shared.lua file following the examples already created there.
  • Create the blue team spawn, the red team spawn, and the center of the map for each map you create

    • Adding images

  • To add the image of the map, just add the image to the folder /img with the same name as the key of the map. (/img/miningyard.jpg) (add jpg files only)
  • To add the cloth images, just create your cloth and take a photo, then upload it like the others already on /img/clothes the image size should be 296 x 407 pixels

Capture the Flag Features

Improve your server with our unique deathmatch mode and create fun events for your community to enjoy.

  • High Optimization : 0.00 ms Idle | 0.01 ms During the game.
  • Scoreboard with Redteam score and Blueteam score displayed during the game.
  • Flag synced to players
  • Add your country Flag or other to the game on shared.lua.
  • Each Team can have up to 10 players, supporting up to 10 vs 10 Game in a deathmatch mode.
  • You can start a match with at Least 1 player in each team
  • When a team gets the pre-defined score (shared.lua), that team wins the game.
  • Timer with countdown during the game
  • If Timer runs out then the team with More score wins the game, if both scores are the same a draw will be called
  • If a player picks the flag and dies, the flag will be dropped on playerdeath location and can be picked by other players, a new position of the flag will be shown on minimap. The player will respawn and can keep playing.
  • When a team delivers the flag on theyr base, will score a point and the flag will return to the original location
  • Ready for QB-Target, Qtarget, bt-target, you can disable it if want the blip system
  • Add / remove / change maps
  • 3 maps Ready : Officeroom / Miningyard / Driftunderground
  • Add or remove all weapons you want to be available on Game.
  • First person joining the lobby will be the host
  • The host can choose on UI the Map and the Weapon for that game.
  • Only the host can start the game
  • The host have time to start the game, if not the party will be disbanded
  • If the host leave the pre-game a new host will be defined
  • Option to close ui and still on teams
  • Option to change teams while the game not started
  • Option to leave game if the game not started
  • Weapon is given to the players on game start and removed when the games end
  • On game start all players inventory gets cleared and given back on game end
  • Option to use the command leavearena, to leave the game while active, weapon got removed and inventory returned to the player
  • Add your custom clothing on config
  • Each player player team can choose the outfit to play on the game from each team. You can add and customize as many as you want
  • Players will be equiped with selected cloth setup when the game starts and back on they’r clothes when game finishes

Paintball Instalation

ESX Framework

    This resource uses escrow Fivem protection. All needed modifications can be made on shared-server / shared-client and shared.lua

    We recomend the use of polyzones, but our script is full compatible without them

    Installing GameWeapon:

  • Extract the folder Paintball and ensure it with the freecity_paintball folder on your server.cfg
  • Add the GameWeapon to your items table in SQL and to Weapons.lua file

    • Make sure when you start the script the NPC Ped is having the paintgun on his hands. If so the GameWeapon is correctly installed.

    Triggers and Shared.lua configuration

  • Make sure to get the correct triggers accordingly to your framework
  • Choose to use from target system or blip system using the vars:
  • Config.spawnMarker
  • Coonfig.useTargetSystem
  • Make sure to check if Config.WeaponAsItem is set to false if you arent using weapons as items.

    • Clothes & Maps

  • Configure the clothes that the players will get when spawning in shared.lua by following our already created examples
  • To create new maps/zones to fight just copy one of the maps created and follow the lines adding the blue team spawn, the red team spawn, and the center of the map
  • If you arent running Polyzones make sure to set the map range accordingly to your map size to make sure the leavecmd works.
  • To add the image of the map, just add the image to the folder /img with the same name as the key of the map. (/img/miningyard.jpg) (add jpg files only)
